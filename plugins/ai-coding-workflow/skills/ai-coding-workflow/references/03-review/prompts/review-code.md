# Prompt: 程式碼審查

## 任務目標

使用 AI 輔助進行程式碼審查，找出問題並提供改進建議。

## 使用時機

- 審查 PR 時
- 自我審查程式碼時
- 需要第二意見時

## Prompt 範本

```prompt
你是一位資深軟體工程師。請審查以下程式碼。

## 變更描述
${CHANGE_DESCRIPTION}

## 程式碼
\`\`\`${LANGUAGE}
${CODE}
\`\`\`

## 審查重點
${REVIEW_FOCUS}（功能正確性 / 效能 / 安全性 / 全部）

## 請審查

### 1. 整體評估

| 項目 | 評分 | 說明 |
|------|------|------|
| 功能正確性 | ⭐⭐⭐⭐⭐ | [說明] |
| 程式碼品質 | ⭐⭐⭐⭐⭐ | [說明] |
| 可維護性 | ⭐⭐⭐⭐⭐ | [說明] |
| 效能 | ⭐⭐⭐⭐⭐ | [說明] |
| 安全性 | ⭐⭐⭐⭐⭐ | [說明] |

**總評**：[一句話總結]

### 2. 問題列表

#### 🔴 必須修改 (Blocker)

| # | 位置 | 問題 | 建議 |
|---|------|------|------|
| 1 | Line X | [問題描述] | [修改建議] |

#### 🟡 建議修改 (Suggestion)

| # | 位置 | 問題 | 建議 |
|---|------|------|------|
| 1 | Line X | [問題描述] | [修改建議] |

#### 🟢 小問題 (Nit)

| # | 位置 | 問題 | 建議 |
|---|------|------|------|
| 1 | Line X | [問題描述] | [修改建議] |

### 3. 優點

- ✅ [做得好的地方 1]
- ✅ [做得好的地方 2]

### 4. 改進後的程式碼

如果有 Blocker，提供修改範例：

\`\`\`${LANGUAGE}
// 修改前
[問題程式碼]

// 修改後
[改進程式碼]
\`\`\`

### 5. 額外建議

- [其他改進建議，如重構、測試等]
```

## 執行後驗證

- [ ] 所有 Blocker 已標示
- [ ] 問題有具體位置
- [ ] 建議具體可執行

## 下一步

- Blocker 修改後重新審查
- 建議項目視情況採納
