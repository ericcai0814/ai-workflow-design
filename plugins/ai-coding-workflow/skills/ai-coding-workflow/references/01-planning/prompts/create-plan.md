# Prompt: 建立執行計畫

## 任務目標

將功能規格轉化為可執行的開發計畫和任務清單。

## 使用時機

- 需求分析完成後
- 開始開發前
- 需要評估工作量時

## Prompt 範本

```prompt
你是一位技術主管。請根據以下功能規格建立執行計畫。

## 功能規格
${FEATURE_SPEC}

## 技術棧
${TECH_STACK}

## 團隊資訊
${TEAM_INFO}

## 請產出

### 1. 任務拆解

| 任務 | 類型 | 相依性 | 估計時間 |
|------|------|--------|----------|
| [任務名稱] | 前端/後端/全端 | [相依的任務] | [時間] |

### 2. 執行順序

```
Phase 1: [階段名稱]
├── 任務 1.1
├── 任務 1.2
└── 任務 1.3 (依賴 1.1)

Phase 2: [階段名稱]
├── 任務 2.1 (依賴 Phase 1)
└── 任務 2.2
```

### 3. 檔案變更預估

| 類型 | 檔案路徑 | 變更內容 |
|------|----------|----------|
| 新增 | src/components/xxx.tsx | 新元件 |
| 修改 | src/pages/xxx.tsx | 加入新元件 |
| 新增 | src/api/xxx.ts | 新 API |

### 4. 風險評估

| 風險 | 可能性 | 影響 | 緩解措施 |
|------|--------|------|----------|
| [風險描述] | 高/中/低 | 高/中/低 | [如何緩解] |

### 5. 驗收檢查點

- [ ] Phase 1 完成：[檢查項目]
- [ ] Phase 2 完成：[檢查項目]
- [ ] 最終驗收：[檢查項目]

### 6. 建議的開發順序

1. 先做：[任務] — 原因：[為什麼先做]
2. 再做：[任務] — 原因：[為什麼]
3. 最後：[任務] — 原因：[為什麼]
```

## 執行後驗證

- [ ] 任務已拆解到可執行的粒度
- [ ] 相依性已標明
- [ ] 有明確的執行順序
- [ ] 風險已識別

## 下一步

- 計畫確認後 → 開始執行第一個任務
- 每個任務完成後 → 執行對應的驗收檢查

## 常見問題

**Q: 任務應該拆到多細？**

A: 每個任務應該可以在 1-4 小時內完成。太大就再拆，太小就合併。

**Q: 估計時間不準確怎麼辦？**

A: 估計時間是參考值，執行過程中持續調整。建議加上 20-30% 緩衝。
